# Load Test — 2026-02-28

## Setup

- **Tool:** Locust 2.43.3
- **Users:** 3 concurrent, ramped at 1/s
- **Duration:** 90 seconds
- **Target:** http://localhost:5000 (Flask dev server + Celery worker, CPU only)
- **Workload:** each user loops — upload PDF → poll `/status` every 2s → download CSV

## Observations

- **OCR is the bottleneck.** Each `process_pdf` task takes 7–9s on CPU. The Celery worker runs
  4 parallel processes (default = CPU count), so effective task throughput is ~0.4–0.5 tasks/s.
- **`/upload` and `/download` are fast.** Flask itself adds negligible overhead — `/upload` just
  saves the file and enqueues (23ms median), `/download` is a file read (3ms median).
- **`/status` p95 is 170ms, p99 is 370ms.** Most polls hit Redis in ~22ms; outliers are from
  Celery updating task state mid-processing which briefly blocks the result backend.
- **Zero failures** after fixing the two bugs below.

## Bugs Found

### 1. Filename collision (race condition)
**Symptom:** ~50% of Celery tasks raised `FileNotFoundError` under concurrent load.
**Cause:** All test users uploaded the same file (`test.pdf`). `secure_filename` does not add
uniqueness, so concurrent uploads overwrote each other in `uploads/`. The first task to complete
deleted the file; subsequent tasks couldn't find it.
**Fix:** Prepended a UUID hex to every saved filename in `main.py`:
```python
filename = f"{uuid.uuid4().hex}_{secure_filename(file.filename)}"
```

### 2. CWD mismatch between Flask and Celery worker
**Symptom:** `/download` returned 404 for every completed job (100% failure rate).
**Cause:** `UPLOAD_FOLDER = 'uploads'` and `OUTPUT_FOLDER = 'outputs'` in `paddlepaddle.py` were
relative paths. Flask (started with `python src/main.py`) and the Celery worker (started with
`celery -A celery_app worker`) resolved them from different working directories, so they were
reading and writing in different places.
**Fix:** Anchored both paths to the `backend/` directory using `__file__`:
```python
_SRC_DIR = os.path.dirname(os.path.abspath(__file__))
_BACKEND_DIR = os.path.dirname(_SRC_DIR)
UPLOAD_FOLDER = os.path.join(_BACKEND_DIR, 'uploads')
OUTPUT_FOLDER = os.path.join(_BACKEND_DIR, 'outputs')
```
